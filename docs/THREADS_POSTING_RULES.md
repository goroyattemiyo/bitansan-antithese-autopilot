# 微炭酸アンチテーゼ｜Threads投稿設計ルール

微炭酸アンチテーゼのThreads投稿では、フォロワー獲得と作品世界への没入を両立させる。

単なる告知やコメント獲得投稿ではなく、投稿単体でも、ねたみ・そねみの物語の一場面として成立させる。

読者が自分の感情を重ねた結果として、反応・フォロー・楽曲視聴につながる設計を目指す。

## 基本方針

投稿では、次の要素を優先的に使用する。

### 型1：感情に賛否や違いが生まれる投稿

読者が、

- わかる
- 自分は違う
- 自分にも経験がある
- ねたみらしい
- そねみらしい

と感じる本音や場面を描く。

扱うテーマ例：

- 嫉妬
- 愛されたい気持ち
- 独占欲
- 強がり
- 承認欲求
- 片思い
- 裏切り
- 推しへの感情
- 友情と依存
- 素直になれない気持ち

強い感情を書く場合も、他人への攻撃ではなく、キャラクター自身の矛盾・弱さ・痛みとして描く。

### 型2：すぐ答えられる質問

読者が長く考えなくても、自分の経験や感覚で答えられる質問を使う。

良い例：

- 嫉妬したら隠す派ですか？
- 会いたいって先に言えますか？
- 元恋人のSNS、見てしまいますか？
- 推しが売れたら嬉しいですか、寂しいですか？
- 好きな人の幸せを素直に喜べますか？

避ける例：

- あなたにとって愛とは何ですか？
- 人間関係についてどう考えますか？
- 幸せの本質とは何でしょうか？

質問は必須ではない。

余韻、二択、キャラクターの独白だけで終える投稿も使用する。

### 型3：感情の一場面

質問を置かず、短い情景や心の動きを描く。

例：

```text
返信が遅いだけなのに、
嫌われた理由を探してしまう。

そねみは、
待つのが苦手なんじゃなくて、
置いていかれるのが怖い。
```

### 型4：キャラクター同士の掛け合い

ねたみとそねみの違いが、一往復程度の短いやり取りで伝わる形にする。

例：

```text
そねみ
「嫌われたかもしれない」

ねたみ
「まだ何も起きてないでしょ」

そねみ
「何も起きてない時間が、一番怖いの」
```

## 投稿の役割

一つの投稿ですべてを達成しようとしない。

投稿ごとに第一目的を一つ決める。

- 新規読者の反応を得る
- ねたみを覚えてもらう
- そねみを覚えてもらう
- 歌詞の世界へ没入してもらう
- キャラクターへの親しみを作る
- 楽曲を聴く理由を作る
- フォローを継続する理由を作る

曲や作品への誘導は、必要な投稿にだけ入れる。

## 基本構成

投稿ごとに、次から3〜5要素を選ぶ。

1. 感情を動かす一行目
2. 具体的な場面
3. ねたみ、またはそねみの本音
4. 本音の裏側にある矛盾や痛み
5. 読者が答えやすい質問または二択
6. 歌詞や楽曲内の感情との接続
7. 余韻のある一文

毎回同じ順番にはしない。

## 楽曲への接続

曲名だけを最後に付けない。

次のいずれかを使って自然につなぐ。

- この感情を歌った曲である
- キャラクターが曲の中で言えなかった本音である
- 歌詞の一場面へ続いている
- 曲を知らなくても投稿単体で成立している

良い例：

```text
これは、
『わたしじゃダメなの？』で
そねみが最後まで言えなかった言葉です。
```

悪い例：

```text
言える派ですか？
言えない派ですか？

『わたしじゃダメなの？』
```

## キャラクター別の使い分け

### ねたみ

扱う感情：

- 嫉妬
- 独占欲
- 負けたくない気持ち
- 攻撃的に見える本音
- 他人の幸せを素直に喜べない感情
- 自分だけを見てほしい気持ち

言葉は強くてもよい。

ただし他人を傷つける主張ではなく、強い言葉の裏にある不安、劣等感、寂しさを残す。

### そねみ

扱う感情：

- 愛されたい気持ち
- 自信のなさ
- 素直になれない気持ち
- 寂しさ
- 傷つくことへの恐れ
- 見捨てられる不安

直接助けを求めず、平気なふりや遠回しな言葉から弱さが見えるようにする。

## 投稿比率の目安

10投稿の場合：

- 賛否や共感が生まれる感情投稿：2本
- すぐ答えられる質問投稿：2本
- 感情の一場面・独白：2本
- 歌詞・楽曲世界の投稿：2本
- キャラクターの日常・掛け合い：1本
- 新曲・動画・作品告知：1本

質問型を全体の半分未満にする。

## 避けること

- 「みんなどう思う？」だけで終える
- キャラクターと無関係な一般恋愛質問
- 毎回二択で終える
- 毎回曲名を付ける
- 同じ冒頭文を繰り返す
- コメントを直接要求する
- 炎上目的の断定
- 他人や属性を攻撃する表現
- 曲の内容と関係のない誘導
- 強い感情投稿を連続させる
- 既存投稿の単純な言い換え量産

## アカウント運用上の安全ルール

- 同型の質問投稿を連続させない
- 同じ曲名やURLを短期間に繰り返さない
- 投稿直後の返信を固定間隔で連投しない
- 同文・類似文を再投稿しない
- 強い主張の間に日常・掛け合い投稿を挟む
- 自動投稿の失敗時に同じ投稿を自動再送しない
- 投稿数を増やすより、内容の違いを優先する

## 投稿作成時のチェック

- 投稿単体で作品として成立しているか
- 誰の言葉か、名前がなくても分かるか
- 読者が感情を重ねられる具体性があるか
- 質問を削っても成立するか
- 質問形式が直近投稿と重複していないか
- キャラクターの人格が出ているか
- 強い本音の裏に弱さや矛盾があるか
- 曲への接続が自然か
- 曲名だけを付けた宣伝になっていないか
- 過去投稿の言い換えになっていないか
- 同じ曲やテーマが続いていないか

## 出力指示

Threads投稿案を作成するときは、次を出力する。

1. 投稿本文
2. 投稿タイプ
3. 担当キャラクター
4. 第一目的
5. 使用した感情テーマ
6. 楽曲または世界観との接続
7. 直近投稿との重複リスク
8. アカウント運用上のリスク
9. 必要な場合のみ別案1本

投稿本文は、宣伝文ではなく、微炭酸アンチテーゼの作品世界の一部として成立させる。

---

## 中心原則

> 質問は反応を得る手段であり、投稿の必須要素ではない。

この原則を中心に、伸ばすための投稿と、キャラクターを好きになってもらう投稿の両方を設計する。
